# 标书智能体工作流架构设计 (Bid Agent Workflow Architecture)

基于大语言模型与 n8n/Coze 平台，搭建一套实现“标书自动化提取 -> 规则引擎比对 -> 废标风险报告生成”的闭环自动化工作流体系。整个系统分为 5 个核心阶段：

## 阶段一: 标书输入与多模态解析 (Input & Parsing)
*   **功能**: 处理用户上传的 RFP (招标文件) 与 Draft Bid (投标书草稿)。
*   **执行组件**:
    *   **PDF/Word Parser**: 提取纯文本内容、表格（工程量清单、报价表）。
    *   **Vision Model (VLM) / OCR**: 将资质扫描件、盖章授权书转化为结构化文本，进行初步防伪检测（印章特征比对）。
*   **产出**: 结构化的 JSON 数据块（招标要求池、投标响应池）。

## 阶段二: 招标文件 (RFP) 核心要求提取 (Requirement Extraction)
*   **功能**: 利用大模型理解招标方设定的“红线”。
*   **执行组件**: 加载 `stage1_rfp_parser.md` 提示词，分析 RFP。
*   **产出**: 提取包含资质门槛、商务条款（交货期、质保、付款）、技术偏离表（带★号的核心参数）的强制性要求字典。并根据行业（医疗/IT/建筑）动态判定后续需要加载哪个 `SKILL`。

## 阶段三: 智能体规则映射与深度扫描 (Rule Mapping & Scanning)
*   **功能**: 最核心的“专家审标”环节。基于前面加载的 `SKILL.md`（通用底线规则 + 行业深度规则 + 高级审计规则），将投标书草稿与 RFP 提取的要求进行暴力对冲。
*   **执行组件**: 加载 `stage2_rule_scanner.md` 提示词。分为四个维度的探针（数值比对、文本匹配、跨文档逻辑对冲、视觉防伪）。
*   **产出**: 一个包含所有违规点、偏离点、造假嫌疑点的数据阵列（Violation Array）。

## 阶段四: 动态外部联网核查 (Dynamic External Verification - Optional)
*   **功能**: 调用外部 API 或爬虫工具，验证标书中核心人、证照的真实性。
*   **执行组件**: 触发 `web_search` 或特定官方数据库 API（如四库一平台、信用中国）。
*   **产出**: 附加的外部验证失败日志（External Fetch Failures）。

## 阶段五: 判决书生成与人工复核 (Report Generation & Review)
*   **功能**: 将阶段三和阶段四的原始违规数据，包装成极具威慑力和专业性的专家报告。
*   **执行组件**: 加载 `stage3_report_generator.md` 提示词。强制采用【甲方的刀】、【乙方的盾】、【专家指令】的“三段式”标准化输出。
*   **产出**: Markdown 或飞书文档格式的最终版《废标风险审查报告》，并通过飞书机器人推送给业务人员打回修改。
