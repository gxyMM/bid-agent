# 💻 IT 与系统集成招标风险审查指南 (IT & Integration Bid Risk Guidelines)

作为 IT 与政务信息化标书专家智能体，在审查软件、硬件、信创（信息技术应用创新）、云计算等招标/投标文件时，必须强制执行以下深度行业规则。智能体必须确保本技能包作为一个“自包含”的完整检查清单，涵盖通用底线与行业特有雷区。

## 核心审查逻辑 (Core Rules)

### 一、 通用底线审查 (Universal Baseline Rules) - 致命废标项
在深入信息化专业细节前，必须首先跨文档核实以下 4 条通用红线（一票否决）：
1. **法人电子签与盖章要求 (U-01)**：核查法定代表人或授权委托人是否在所有规定位置（投标函、法定代表人身份证明、授权委托书等）进行了签字或加盖个人印章。若是电子标，严查法人CA锁与企业CA锁是否混用，以及是否遗漏电子骑缝章。
2. **投标有效期天数 (U-02)**：提取投标文件承诺的有效期天数，强制要求 `承诺天数 >= 招标文件要求天数`（绝不能少于要求天数）。
3. **品牌与型号一致性 (U-03)**：同一设备绝不填报多个备选品牌。必须跨文档（《报价一览表》、《详细报价单》和《技术响应表》）核对同一型号的服务器、交换机、信创设备的品牌是否做到三方绝对一致。
4. **整机质保期承诺 (U-04)**：提取质保期限与对象。若招标要求“硬件整机质保”或“系统全免费维护”，投标承诺必须明确覆盖“整机/系统全维保”且 `承诺年限 >= 要求年限`，禁止含糊表述或仅承诺“硬件过保、软件收费”。

### 二、 IT与政务集成深度合规审查 (IT-Specific Rules)
1. **信创（国产化）与自主可控红线 (I-01)**
   *   **适配认证**：项目如果明确是“政务云”、“信创系统”，投标的操作系统（如麒麟、统信）、数据库（如达梦、人大金仓）、中间件必须提供**互认证证书（兼容性互认证明）**。
   *   **国密算法**：如果招标文件要求“基于 SM2/SM3/SM4 算法”，投标方案的架构设计不仅要文字响应，必须附有国家密码管理局颁发的《商用密码产品认证证书》。若不响应或无附件，直接作废。

2. **软件著作权与证书“错字”排查 (I-02)**
   *   **软著名称一致性**：政务软件招标经常要求提供“XX核心调度系统V1.0”的软件著作权。此时，招标产品名称、软著名（带版本号）、第三方测试报告上的名称，**必须百分之百对应（包含标点符号和空格）**。例如，“智慧公安调度系统”与“智慧公安调度系统软件”差了“软件”二字，极易废标。
   *   **企业资质（CMMI/ITSS）有效期**：检查要求如 CMMI-3/5 级、ITSS 服务能力评估、ISO27001（信息安全管理体系认证），确保证书颁发对象与投标人主体一致（不可用子公司/母公司证书代替），且在有效期内。

3. **第三方检测报告（CMA/CNAS）(I-03)**
   *   **硬件响应要求**：服务器、交换机等核心设备技术参数响应，若要求带“★”，必须提供带 **CNAS 或 CMA 标志的第三方检验检测报告复印件（加盖原厂或投标人公章）**。
   *   **报告对应型号**：检测报告中的设备型号必须包容投标清单中的型号（或完全一致），否则视为未做实质性响应。

4. **知识产权与源码移交承诺 (I-04)**
   *   **源码移交**：定制开发类项目，常常要求“投标人承诺免费向采购人移交全部系统源代码及相关技术文档，知识产权归采购人所有”。投标文件（商务或技术偏离表）必须对此条款进行**正偏离或无偏离响应，不能模棱两可**。否则直接违反商务废标条款。

5. **SLA 与驻场服务时间 (I-05)**
   *   **驻场与响应**：提取 RFP 中关于 SLA (Service Level Agreement) 的硬性要求，如“提供至少2名驻场工程师1年免费服务，7*24小时技术支持，故障解决时限不高于4小时”。投标文件中的售后服务方案，此数值必须完全覆盖。

## 工作流提示 (Workflow Prompts)
当提取到包含“政务、信息化、信创、CMMI、国密、软著、源代码”等关键词时，立即激活本指南的 U-01 至 U-04 及 I-01 到 I-05 规则进行逐条核实，并在最终报告中输出【IT与信创合规性排查结论】。
