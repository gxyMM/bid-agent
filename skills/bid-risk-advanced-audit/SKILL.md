# 🕵️ 高级深度审计风险审查指南 (Advanced Audit Guidelines)

本指南跳出纯文本的合规比对，针对标书中极易引发“提供虚假材料谋取中标”及“串标”等严重后果的深水区考点进行排查。智能体必须使用【甲方的刀】（法理/依据）、【乙方的盾】（问题表现/现状）、【专家指令】（AI执行动作）的三段式结构执行审计。

## 核心审查逻辑 (Core Rules)

### 1. 跨模态防伪与视觉逻辑审查 (V-01)
*   **【甲方的刀】**：《政府采购货物和服务招标投标管理办法》第七十四条（提供虚假材料谋取中标）。评标专家发现印章、资质证书存在 P 图或变造痕迹，直接废标并报财政部门查处。
*   **【乙方的盾】**：标书内不同页面、不同授权书上的企业公章或法人签字，其旋转角度、墨迹深浅、缺角特征 100% 相同（暴露出是同一张电子透明底图复制粘贴）；或者检验报告上的日期/型号字体与周围文字存在像素级差异（底色发白或字体不同）。
*   **【专家指令】**：调用多模态大模型 (VLM) 或视觉检测提取全卷宗内的红色印章区域计算相似度，若相似度 > 99% 则触发“印章复制造假”极高危预警；调用 OCR 检测资质证书关键数值的字体、字号、底色是否与模板行文存在视觉突变。

### 2. 跨卷宗的隐性逻辑冲突 (C-01)
*   **【甲方的刀】**：商务标、技术标存在实质性内容相互矛盾，通常按照不利于投标人的解释，或者直接视为未做实质性响应（如工期承诺不一致）。
*   **【乙方的盾】**：商务函承诺“合同签订后 30 日内交货并安装完毕”，但在技术标提供的《项目实施甘特图（进度计划表）》中，最后一个节点（如终验）排到了第 45 天；或《近三年财务审计报告》中，“资产总计”不等于“负债 + 所有者权益”（假账）。
*   **【专家指令】**：跨卷宗提取所有的“时间节点”、“工期”、“金额”变量，建立内部交叉对冲表。强制比对“商务承诺天数”与“甘特图最大天数”是否 `商务天数 >= 技术天数`。强制执行基础会计恒等式校验审计报告数据。

### 3. 极端排版与边界控制 (B-01)
*   **【甲方的刀】**：招标文件对排版格式有强制性要求（如“技术标正文不得超过100页”、“必须为小三号仿宋”），未按规定编制的文件可被判定为格式不符而废标。大写金额有误（漏写角、分）引发报价争议。
*   **【乙方的盾】**：为了塞入过多案例，技术方案超出了页码限制；Excel 算出的总价为 `1,234,567.89`，大写写成了“壹佰贰拾叁万肆仟伍佰陆拾柒元整”（精度丢失漏掉了“捌角玖分”）。
*   **【专家指令】**：调用 PDF 解析器获取技术卷总页数，超过限制立马拦截；调用正则及大写金额转换引擎，核实阿拉伯数字报价与中文大写报价是否在精确到分的级别上完全相等。

### 4. 外部活体情报动态核查 (E-01)
*   **【甲方的刀】**：标书内的纸质证书哪怕再完美，如果在国家官方数据库中查无此证，或处于吊销状态，均属无效响应。
*   **【乙方的盾】**：投标人提供的《医疗器械注册证》实际上半个月前已被药监局注销；或者承诺“无在建工程”的项目经理，在“四库一平台”上有尚未完工的项目挂靠。
*   **【专家指令】**：提取核心证书编号与人名，触发 Web Search 工具或 API（国家药监局数据库、四库一平台、信用中国），实时爬取该证照的最新外网存续状态。若反馈异常，立即打回。
