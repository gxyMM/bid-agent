# 标书风控 Agent 技术方案白皮书 (V1.0)

## 1. 业务背景与价值
在招投标领域，由于标书文件篇幅长（通常 100+ 页）、规则严苛、法律后果重，人工审计极易产生疲劳导致漏看致命废标项。本方案通过 AI Agent 构建一套“招标文件解读 + 投标文件风险评估”的双向审计系统，旨在实现“上传即排雷”的终极风控体验。

## 2. 核心技术架构

### 2.1 全链路工作流
1. **入口端**：通过飞书表单/插件上传《招标文件 (RFP)》与《拟投标文件 (Draft)》。
2. **触发端**：飞书自动化工作流触发 Webhook 回调至后台审计引擎。
3. **处理端 (Core Engine)**：
    - **文档感知**：基于 `PyMuPDF` 进行视觉坐标探测。
    - **页码对齐**：建立“物理-逻辑”双轴映射，确保回填证据页码与纸质打印版完全一致。
    - **语义检索**：根据行业 Skill 清单，对招标文件和投标文件进行交叉 RAG 检索。
    - **风险判定**：利用 LLM 判定响应偏离度，生成专家级修复建议。
4. **回填端**：将结果自动同步至飞书多维表“风控终审驾驶舱”。

### 2.2 关键黑科技：视觉坐标对齐算法
针对 PDF 文档常见的“三无”（无 PageLabels 元数据）问题，系统采用 **Footer 区域探测技术**：
- 锁定页面底部 80 像素区域。
- 提取并正则宽容匹配 `第\s*(\d+)\s*页`。
- 动态修正 PDF 物理序号与文档逻辑页码的偏移量，解决“证据页码对不上”的信任危机。

### 2.3 双屏交互视觉引擎 (Advanced Visualization)
为了解决长文档“溯源难、核对累”的问题，系统引入飞书多维表插件（Base Extension）实现双屏比对：
- **分屏布局**：采用 Dashboard Extension 架构，左右 1:1 渲染招标文件与拟投标文件。
- **坐标高亮**：前端利用 `pdf.js` 结合后端返回的 `coordinates` 坐标，在 Canvas 上动态绘制半透明遮罩，实现一键定位地雷点。
- **数据联动**：实时监听 Bitable 行选中事件，实现“选中即定位，定位即高亮”。

## 3. 批量化处理能力
### 3.1 批量附件上传工作流
针对大规模投标场景，系统提供独立的批量上传工具：
1. **分块上传 (Chunk Upload)**：支持 20MB 以上超大标书稳定同步至飞书云空间。
2. **记录聚合**：自动解析文件夹结构，将同项目的招标文件与投标文件聚合为多维表中的单一 Record 任务。
3. **QPS 削峰**：内置频率控制器，确保在批量导入时不会触发飞书开放平台的 API 限流。
系统内置三大行业专用 Skill 模块：
- **医疗设备**：像素级型号比对、★项原厂图证溯源、授权链完整性追踪。
- **IT 与系统集成**：源代码权属红线、国密/信创证书校验、驻场社保主体核实。
- **建筑工程**：规费/税金/不可竞争费率红线校验、四库一平台在建工程联动。

## 4. 数据资产与输出规范
每一条风险诊断严格遵循证据链闭环：
- **【甲方的刀】**：招标文件原文 + 精准页码。
- **【乙方的盾】**：投标文件响应现状描述 + 精准页码。
- **【补缺指令】**：明确的、可执行的整改操作建议。

## 5. 未来演进
- **多模态图纸审计**：支持对建筑 CAD/设备图纸中的微小文字标识进行合规性扫描。
- **全网数据对标**：联动招标公告网，实时核实业绩合同的真伪与中标价参考。
